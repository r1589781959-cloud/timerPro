## 启动方式

当前服务器不是 Docker 部署，Docker 相关文件不是本阶段依据。实际以服务器终端直接运行 Python 为准。

主应用默认端口为 `5050`，也支持用环境变量覆盖：

```powershell
cd E:\antigravity_project_01
$env:PORT="5050"
python web_app/main_saas.py
```

也可以显式用 uvicorn 指定任意端口：

```powershell
cd E:\antigravity_project_01
python -m uvicorn web_app.main_saas:app --host 0.0.0.0 --port 5050
```

超级管理后台默认端口为 `5051`，支持 `ADMIN_PORT` 覆盖：

```powershell
cd E:\antigravity_project_01
$env:ADMIN_PORT="5051"
python web_app/admin_dashboard.py
```

超级管理后台使用 HTTP Basic Auth，账号密码来自环境变量：

- `ADMIN_USER`，默认 `admin`
- `ADMIN_PASS`，默认 `timerPro@2026`

## 顾客端号牌方案当前状态

最新业务决策：

- 每个门店有一个“本店通用顾客入口”二维码/链接。
- 顾客扫本店通用入口后，只输入自己的号牌/标识，不需要也不应该手动输入门店编号。
- 商家仍可预先生成多个固定号牌/NFC 链接，例如 1-50；这些链接用于省略顾客输入号牌的步骤。
- 底层仍统一走顾客查单逻辑：门店 + 号牌找到当前未结订单，再通过 `group_id` 展示同行订单。
- 多人同行保留“一个顾客一个号牌”，同行订单共用 `group_id`。
- 多人团购根据团购配置 `persons` 自动调整开台人数和号牌输入框数量。

入口形式：

```text
/customer.html?shop=<shop_code>
/customer.html?shop_code=<shop_code>
/customer.html?s=<shop_code>
```

顾客页只显示号牌输入框。若链接没有门店信息，会提示顾客扫描店内专属顾客入口码。

固定号牌/NFC 直达入口：

```text
/customer.html?t=<token>
```

## 顾客端相关接口

公开接口：

```text
GET /api/customer/by-code?shop_code=<shop_code>&label=<label>
GET /api/customer/by-token/{token}
```

商家端接口，需登录：

```text
GET  /api/customer-codes
POST /api/customer-codes/batch
GET  /api/customer-codes/export
```

`GET /api/customer-codes` 和 `POST /api/customer-codes/batch` 当前都会返回：

- `shop_code`
- `universal_url`：本店通用顾客入口
- `codes[]`：固定号牌/NFC 直达链接列表

`POST /api/customer-codes/batch` 支持两种生成方式：

- 连续编号：`prefix + start/end`
- 自定义号牌：`labels: ["A01", "小熊", "VIP-1"]`

`GET /api/customer-codes/export` 支持导出二维码、NFC 文件或整包：

```text
scope=all|universal|badges
asset=bundle|qr|nfc
include_label=0|1
```

导出规则：

- `asset=qr`：导出二维码；单个通用入口会直接返回 PNG，多个号牌返回 ZIP。
- `include_label=1`：二维码底部会附带入口/号牌标识，方便直接打印。
- `asset=nfc`：导出 NFC 写入文件 ZIP，每个入口/号牌各一个 `.txt`，文件内容就是要写入 NFC 标签的 URL。
- `asset=bundle`：同时包含 `qrcodes/` 与 `nfc/` 两个目录。

商家端设置页只展示“本店通用顾客入口”和操作按钮，具体号牌生成/导出放在弹窗里，避免设置页过长。

## 已完成的重要改动

- 登录框从邮箱专用输入改为“邮箱或用户名”文本输入。
- 订单 ID 改为 UUID，降低多端/多商家冲突风险。
- 订单接口加强按当前商家过滤，避免跨商家数据误读/误操作。
- 活跃订单和顾客端刷新改为静默刷新，避免整个界面一闪一闪。
- 顾客端返回信息扩充，包含模式、已用时间、剩余/超时时间、预估费用、团购/加时明细、同行订单汇总等。
- 开台支持 `identifiers`，多人同行会创建多条订单并共享 `group_id`。
- 设置页支持批量生成顾客号牌码。
- 号牌码支持自定义文本批量生成，并支持一键导出带/不带标识的二维码、独立 NFC URL 文本文件和完整物料包。
- 超级管理后台 `web_app/admin_dashboard.py` 已存在，可查看商家、用户、订单、历史和数据库概况。
