# 📂 TimerPro SaaS 项目结构解析

目前 `web_app` 目录采用的是 FastAPI 架构，以下是各文件的具体分工：

## 1. 核心启动与基础配置
*   **`main_saas.py`**: **SaaS 版的总引擎**。所有的 API 路由（订单、设置、商家中心）都在这里挂载，同时负责跨域（CORS）、静态资源分发以及强效缓存控制。
*   **`database.py`**: **数据库心脏**。定义了所有的数据库模型（`Order`, `Shop`, `SystemConfig`, `GroupBuy` 等）。
*   **`auth.py`**: **鉴权中心**。负责商家的登录验证、JWT 令牌生成以及身份提取（`get_current_shop`）。

## 2. 业务功能路由 (`routers/` 目录)
*   **`routers/orders.py`**: **业务中枢**。处理“开台”、“实时计费”、“结账”、“订单列表”等功能。
*   **`routers/merchants.py`**: **商家管理**。处理商家的注册、登录、个人资料修改。

## 3. 静态资源与前端 (`static/` 目录)
*   **`static/index.html`**: **前端 UI 界面**。

1.  **暂停时长抵扣**：后台虽然有暂停按钮，但计费引擎还未真实扣除暂停期间产生的费用（核心逻辑在开发中）。
