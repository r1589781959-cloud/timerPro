## 概述

本报告详细验证了时段团购模式(`time_slot`)的所有功能和边界情况。时段团购是一种限定时间窗口的团购产品，仅在指定时段内开始才能享受团购价格，其他时段则自动转为散客计费。

| **超时价格** | ¥10.9 /小时 |
| **缓冲期** | 10分钟 |
| **人数** | 1人 |
| **计费模式** | 分阶计费 (10分钟 @ ¥2.0) |

## 核心机制说明

### 1️⃣ 时段判定

- ✓ **时段内开始** (10:00 ≤ 开始时间 ≤ 14:00) → 按时段团购计费
- ✗ **时段外开始** (开始时间 < 10:00 或 > 14:00) → **禁止开台**
- ✓ **边界处理** (10:00和14:00边界) → 包含边界,视为有效时段

### 2️⃣ 时段内计费逻辑

1. **实际可用时间** = min(定额时长, 时段剩余时间)
   - 例如：13:00开台，定额240分钟，但时段剩余60分钟 → 实际可用60分钟

2. **超时使用** (> 实际可用时间)
   - 超时时长 = 使用时长 - 实际可用时间
   - 缓冲 = 10分钟
     - 若 超时时长 ≤ 10分钟 → 无需支付超时费
     - 若 超时时长 > 10分钟 → 按分阶计费收取超时费

3. **超时费计算** (分阶模式)
   - 周期 = 10分钟
   - 周期费 = ¥2.0
   - 14分钟超时 → 2个周期 → ¥4.0
   - 60分钟超时 → 1小时 → ¥10.9
   - 135分钟超时 → 2.25小时 → ¥24.5

### 3️⃣ 核销状态影响

| 状态 | 有超时费 | 无超时费 |
|------|---------|---------|
| ✗ 未核销 | 应收 = 券价 + 超时费 | 应收 = 券价 |
| ✓ 已核销 | 应收 = 仅超时费 | 应收 = ¥0.0 |

**说明**: 核销后只需支付超时费,券价已由商家承担

### 4️⃣ 时段外计费逻辑(散客转换)

- 自动使用先玩后付模式
- 起步: 60分钟 @ ¥10.9
- 缓冲: 10分钟
- 超时费: (超时分钟 / 60) × ¥10.9

## 核销逻辑详解

时段团购的两种收费模式:

1. **未核销**(消费者全额支付)

   ```
   应收 = 自选定额券 + 超时费
   ```

2. **已核销**(商家补贴券价)
