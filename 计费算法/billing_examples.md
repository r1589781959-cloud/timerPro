# TimerPro 阶梯计费算法示例（完整参数覆盖）

## 📋 当前参数

| 参数 | 值 | 含义 |
|------|------|------|
| `price_base` | **10.9** 元 | 基准时长内的固定费用 |
| `time_base` | **60** 分钟 | 基准时长（price_base 覆盖的范围） |
| `price_overtime` | **10.9** 元/小时 | 超时每小时封顶价（也是整小时单价） |
| `buffer_min` | **10** 分钟 | 超时宽限期（此内免费） |
| `calc_mode` | **step** | 阶梯计价 |
| `step_n` (N) | **10** 分钟 | 阶梯块大小 |
| `step_y` (Y) | **2** 元 | 每块费用 |
| `step_k` (K) | **2** | 进位分割系数。余数 ≥ N/K 时进一块 |
| `ceil_x` (X) | **5** 分钟 | 抹零界限。距下一整小时 ≤ X 分钟时，按整小时收 |
| `price_unlimited` | **59.9** 元 | 全天畅玩价 |
| `price_single_board` | **39.9** 元 | 单板不限时价 |

## 🔧 阶梯计价核心算法 `get_overtime_cost(over_mins)`

```
输入: 超时分钟数 over_mins
输出: 费用

1. hrs = over_mins ÷ 60 (取整)     → 整小时部分
2. rem = over_mins % 60             → 不足一小时的零头
3. 计算零头费用 rem_cost:
   a. 如果 (60 - rem) ≤ ceil_x(5)  → 差5分钟就满一小时 → rem_cost = price_overtime(10.9)
   b. 否则:
      blocks = rem ÷ step_n(10)    → 完整块数
      if (rem % N) ≥ (N ÷ K)      → 即余数 ≥ 5 → blocks + 1
      rem_cost = blocks × step_y(2)
      if rem_cost > price_overtime → rem_cost = price_overtime (封顶)
4. 总费用 = hrs × price_overtime + rem_cost
```

## ⚡ 关键参数边界案例汇总

> 以下展示每个参数在什么场景下**改变了计费结果**。

### N (step_n=10) 的作用
```
超13分: 13÷10=1整块, 余3 → 3<5不进位 → 1×2 = ¥2
超15分: 15÷10=1整块, 余5 → 5≥5进位  → 2×2 = ¥4   ← 余数够半块就多收一块
超20分: 20÷10=2整块, 余0            → 2×2 = ¥4
```

### K (step_k=2) 的作用
```
N/K = 10/2 = 5 → "半块"进位阈值
超4分的零头: 4 < 5 → 不进位 → ¥0
超5分的零头: 5 ≥ 5 → 进1位 → ¥2   ← K决定了"多少零头算一块"
```

### X (ceil_x=5) 的作用
```
超54分: 60-54=6 > 5 → 正常算: 5块×2=¥10
超55分: 60-55=5 ≤ 5 → ⚡抹零: 直接按整小时 = ¥10.9  ← 差5分钟就够一小时，补齐
超59分: 60-59=1 ≤ 5 → ⚡抹零: ¥10.9
```

### buffer_min (=10) 的作用
```
定额60分, 玩了68分: 超8分 ≤ 10 → ¥0免费     ← 超时在宽限内不收费
定额60分, 玩了72分: 超12分 > 10 → step(12)=¥2 ← 超宽限才开始计费
```

### price_overtime (=10.9) 的封顶作用
```
超45分: 5块×2=¥10 < 10.9 → 正常
超55分: ⚡抹零 → ¥10.9 (封顶)
如果 step_y 很大(比如5元), 3块×5=¥15 > 10.9 → 封顶为 ¥10.9   ← 永远不超过每小时单价
```
